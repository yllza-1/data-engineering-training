# Day 7 - SQL Detective Day

## Overview

The goal of this practice was to find and fix SQL query errors, validate the results using SQL, and create a business report based on verified data. This helped improve SQL debugging, JOINs, aggregation, and business reporting skills.

## Files Included

- `setup.sql` – Creates and populates the database.
- `table_inspection.sql` – Inspects the database tables and data.
- `fixed_queries.sql` – Contains the corrected SQL queries.
- `validation_queries.sql` – Contains queries used to verify the business report.
- `verified_business_report.md` – Business report based on validated SQL results.
- `query_explanations.md` – Explains the fixed and validation queries.
- `README.md` – Project documentation.

## How to Run

Run the files in the following order:

1. `setup.sql`
2. `table_inspection.sql`
3. `fixed_queries.sql`
4. `validation_queries.sql`
5. Review `verified_business_report.md`

## What I Learned About Debugging SQL

- How to identify and fix SQL syntax errors.
- How to use JOIN correctly to access data from related tables.
- How to use GROUP BY, HAVING, and ORDER BY correctly.
- How to fix queries by checking table structure and column names.

## What I Learned About Verifying Business Reports

- Every business insight should be supported by a SQL query.
- Reports should only use verified results from the database.
- Validation queries help ensure that business reports are accurate and reliable.
